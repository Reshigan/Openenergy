Best when: operators live in one journey all day and want density and zero context loss. Cost: the canvas can get tall; needs disciplined progressive disclosure.

Direction B · the bet

Type it into existence. The journey stays put.

A command bar is the spine of the whole product. New project starts by typing (or ⌘K); a composer drawer slides in over the current journey, which stays visible and dimmed behind it. You never leave your place. Every create, search and jump is one keystroke, so the Atlas becomes the command bar.

Best when: power users move fast across many journeys and hate mousing. Cost: keyboard-first needs discoverable hints for new users; the drawer must never hide critical context.

Direction C · the bet

One thing at a time. The system knows the next step.

The screen always shows the single most important thing to do, full attention, with the journey as a slim context rail. Creating a project is a native guided flow that feels the same as advancing a case: big inputs, plain language, one decision per screen, and a clear "what happens next." Onboarding and daily work use the identical shape.

Best when: occasional or mixed-expertise users, high-consequence steps, lowest cognitive load. Cost: power users may find it slow; needs a fast lane for experts.
