# Open Energy Platform — Product Context

## Register
product

## Product Purpose
South African energy exchange: power trading, carbon credit lifecycle, IPP project management, settlement, regulatory compliance, and O&M operations. Deployed at oe.vantax.co.za. Aligned to ERA 2006, NERSA Grid Code, POPIA, Carbon Tax Act, REIPPPP, JSE-SRL.

## Users

| Role | Primary job | Key stress points |
|---|---|---|
| Trader | Buy/sell energy contracts, manage risk, report to FSCA | Real-time positions, SLA breaches, margin calls |
| IPP Developer | Run project lifecycle from REIPPPP bid to COD | Milestone slippage, stage gate blockers, bond expiry |
| Lender | Underwrite and monitor project finance | Covenant breaches, DSCR, drawdown gating |
| Offtaker | Manage PPA contracts, take-or-pay, RECs | Curtailment claims, tariff CPI, payment security |
| Grid Operator | Dispatch, wheeling, ancillary services | Load shed, dispatch nominations, grid code compliance |
| Carbon Fund | MRV, Article 6, PCAF financed emissions | Verification state, reversal risk, TCFD disclosure |
| Regulator (NERSA) | Licensing, inspections, enforcement, tariff | Inbox overflow, SLA exposure, levy collection |
| ESCO / O&M | Work orders, PM compliance, permit-to-work | Availability guarantees, asset health, HSE incidents |
| EPC Contractor | Submittals, RFIs, ITPs, site diary | Punch list, NCRs, method statement approvals |
| Admin | Platform ops, KYC, cascade DLQ, ML | System health, tenant ops |
| Support | ITIL tickets, problem management | P1 SLA countdown, escalation paths |

## Brand Tone
Institutional, high-stakes, professional. The platform operates at the intersection of capital markets and critical national infrastructure. No gamification, no casual language. Think Bloomberg meets environmental compliance bureau.

## Anti-References
- SAP Fiori (current shell — too rigid, too much sidebar scroll)
- Generic SaaS dashboards (cookie-cutter hero metric cards)
- Bloomberg Terminal raw aesthetic (too dark, too dense for 9 distinct roles)
- Material Design admin panels (too consumer-grade)
- Bright fintech apps (Robinhood/Revolut aesthetic — wrong register)

## Strategic Principles
1. Every screen surfaces what matters now (urgency-first, not alphabetical)
2. Role context shapes the entire workspace — not just the sidebar
3. Data density is a feature, not a problem
4. AI assists are inline and contextual — never a tab or modal gate
5. Regulatory evidence chain is always one click away
6. State machine transitions are first-class UI — not buried in tables

## Success Criteria
- Professional first impression that beats category leaders (Bloomberg, NTT, Procore, E-CARM)
- Operators find their most urgent items within 5 seconds of login
- No dead ends — every screen links to the action that resolves its data
- Mobile-capable for site visits (O&M, EPC roles)
