# Do-Next Stream + Per-Role Depth Audit Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Turn the existing `/cockpit` JourneyCockpit into a do-next stream — one attention-ranked card feed where journeys demote to in-place filter chips, the dead stage-rail becomes a filter control, and each card carries an insight line — then pilot it on carbon_fund and generate a truthful per-role depth matrix.

**Architecture:** Frontend-only rearrangement of data the cockpit already fetches. `GET /api/horizon/:role` already returns `{ lanes, duty, counts }` with cases carrying `bucket / status / score / quantum_zar / actions`. `ItemCard` already promotes `actions[0]` via `PrimaryAction`. The remaining work: (1) journey tabs → filter chips, (2) wire the no-`onClick` stage-rail to set the active chip, (3) add a one-line insight + sparkline to each card, (4) validate the carbon_fund pilot surfaces feed cards, (5) run a mechanical read-only depth audit generating a regenerable matrix doc. No backend change. No new query paths. The `MERIDIAN_CHAINS` security invariant is untouched.

**Tech Stack:** React + TypeScript SPA (Vite), Tailwind classes, Playwright browser tests, vitest. All under `open-energy-platform/pages/`.

## Global Constraints

- **No backend change.** The stream reads `horizon/:role` output and calls existing transition endpoints only. Copied verbatim from spec §2.
- **Security invariant (load-bearing):** SQL identifiers come only from the `MERIDIAN_CHAINS` static literal, never request input; request values bind to `?` placeholders only. The stream introduces no new query path. Copied verbatim from spec §4.
- **Insight is derived, no new fetch.** The per-card insight line + sparkline derive from the case's existing `score / quantum_zar / bucket`. Spec §1.1.
- **Rate-limiter discipline:** any browser test seeds token via `page.addInitScript(localStorage.setItem('token', ...))` with one API login (10 logins / 5 min / IP). Spec §Testing.
- **Depth-audit honesty rule:** only carbon_fund is audited from files read. The other 9 roles' L-levels are generated by running the audit pass over `roleData` + `MERIDIAN_CHAINS` + `surfaces.tsx` — never guessed. Spec §"Honesty rule".
- **Pilot role:** carbon_fund. Its surfaces: `carbon_fund:mrv` (MrvSurface), `carbon_fund:vintages` (VintagesSurface), `carbon_fund:certificates` (CertificatesSurface). Spec §3.
- **Demo login:** password `Demo@2024!`, email `carbon@openenergy.co.za`.

```tsx
// open-energy-platform/pages/src/meridian/StreamInsight.tsx
// Do-next stream: one-line "why me / why now" insight + a derived sparkline.
// Pure derivation from the case's existing score / quantum_zar / bucket — NO fetch.
import React from 'react';

type StreamCase = { bucket?: string; status?: string; score?: number; quantum_zar?: number };

function randShort(zar: number): string {
  if (zar >= 1_000_000) return `R${(zar / 1_000_000).toFixed(1)}m`;
  if (zar >= 1_000) return `R${(zar / 1_000).toFixed(0)}k`;
  return `R${zar.toFixed(0)}`;
}

export function insightLine(c: StreamCase): string {
  if (c.quantum_zar && c.quantum_zar > 0) return `${randShort(c.quantum_zar)} at stake · needs your call`;
  if ((c.score ?? 0) >= 80) return 'High attention — top of queue';
  if (c.status) return `Waiting: ${String(c.status).replace(/_/g, ' ')}`;
  return 'Ready for next step';
}

// Deterministic 8-point series: a gentle ramp toward the score. No random (breaks resume + tests).
export function sparklinePoints(score: number): number[] {
  const s = Math.max(0, Math.min(100, score || 0));
  return Array.from({ length: 8 }, (_, i) => Math.round(s * (0.4 + (0.6 * i) / 7)));
}

export default function StreamInsight({ c }: { c: StreamCase }): JSX.Element {
  const pts = sparklinePoints(c.score ?? 0);
  const max = Math.max(1, ...pts);
  const d = pts.map((p, i) => `${(i / 7) * 60},${16 - (p / max) * 14}`).join(' ');
  return (
    <div className="jc-insight">
      <span className="jc-insight-txt">{insightLine(c)}</span>
      <svg width="60" height="16" viewBox="0 0 60 16" className="jc-spark" aria-hidden="true">
        <polyline points={d} fill="none" stroke="currentColor" strokeWidth="1.5" />
      </svg>
    </div>
  );
}
```

**Files:**

**Interfaces:**
- Consumes: `pages/src/ux-alternatives/launchpad-nav/roleData.ts` (roles → domains → features), `src/utils/chain-registry-meridian.ts` (`MERIDIAN_CHAINS`), `pages/src/meridian/surfaces.tsx` (`SURFACE_REGISTRY`).
- Produces: a role × journey × current-L × card-readiness matrix with a per-row verdict (`journey-ready` / `thin-card` / `dead-tile`), written to `docs/operations/ROLE_DEPTH_MATRIX.md`. Regenerable by re-running the script.

**Method (from spec §"Method"):** per role, walk `getRoleConfig(role).domains → features`; classify each by derivable signals — `chainKey` in `MERIDIAN_CHAINS` + lanes + actions → L3 candidate; Bucket-B surface (registered in `SURFACE_REGISTRY`, no descriptor) → L2–L3; tile resolves to nothing → L1 (dead-tile); transition route on `fireCascade` + SLA sweep → L4; certified export/reconcile → L5. Only signals derivable from the three source files — no guessing.

- [ ] **Step 1: Write the audit script**

```js
// open-energy-platform/scripts/audit-role-depth.mjs
// Mechanical, read-only depth audit. Reads roleData + MERIDIAN_CHAINS + SURFACE_REGISTRY
// and emits docs/operations/ROLE_DEPTH_MATRIX.md. No opinion, no guessing — every
// L-level is derived from a signal present in these three files (spec honesty rule).
import { readFileSync, writeFileSync } from 'node:fs';
import { fileURLToPath } from 'node:url';
import { dirname, resolve } from 'node:path';

const here = dirname(fileURLToPath(import.meta.url));
const R = (p) => resolve(here, '..', p);

const roleData = readFileSync(R('pages/src/ux-alternatives/launchpad-nav/roleData.ts'), 'utf8');
const chains = readFileSync(R('src/utils/chain-registry-meridian.ts'), 'utf8');
const surfaces = readFileSync(R('pages/src/meridian/surfaces.tsx'), 'utf8');

// Chain keys present in MERIDIAN_CHAINS (L3 candidate: state machine + lanes + actions).
const chainKeys = new Set([...chains.matchAll(/['"]([a-z0-9_]+)['"]\s*:/gi)].map((m) => m[1]));
// Surface keys registered as '<role>:<key>' (Bucket-B: L2–L3).
const surfaceKeys = new Set([...surfaces.matchAll(/['"]([a-z0-9_]+:[a-z0-9_]+)['"]/gi)].map((m) => m[1]));

// Parse role → features from roleData (feature objects carry `key` + optional `chainKey`/`route`).
// Group by the nearest role literal above each feature block.
const roleBlocks = [...roleData.matchAll(/role:\s*['"]([a-z_]+)['"]([\s\S]*?)(?=role:\s*['"]|$)/gi)];

function classify(role, featKey, hasChainKey) {
  const chained = hasChainKey && chainKeys.has(hasChainKey);
  const asSurface = surfaceKeys.has(`${role}:${featKey}`);
  if (chained) return { L: 'L3', ready: 'journey-ready' };
  if (asSurface) return { L: 'L2–L3', ready: 'thin-card' };
  return { L: 'L1', ready: 'dead-tile' };
}

const rows = [];
for (const [, role, body] of roleBlocks) {
  const feats = [...body.matchAll(/key:\s*['"]([a-z0-9_]+)['"](?:[^}]*?chainKey:\s*['"]([a-z0-9_]+)['"])?/gi)];
  for (const [, featKey, chainKey] of feats) {
    const { L, ready } = classify(role, featKey, chainKey);
    rows.push({ role, feature: featKey, L, ready });
  }
}

const byRole = {};
for (const r of rows) (byRole[r.role] ||= []).push(r);

let md = `# Role Depth Matrix\n\n`;
md += `> Generated by \`scripts/audit-role-depth.mjs\`. Regenerable — do not hand-edit.\n`;
md += `> Method: spec \`docs/superpowers/specs/2026-07-08-do-next-stream-design.md\` §"Method".\n`;
md += `> Signals are derived from roleData + MERIDIAN_CHAINS + SURFACE_REGISTRY only. No guessed L-levels.\n\n`;
md += `| Role | Feature | Current-L | Verdict |\n|---|---|---|---|\n`;
for (const role of Object.keys(byRole).sort()) {
  for (const r of byRole[role]) md += `| ${r.role} | ${r.feature} | ${r.L} | ${r.ready} |\n`;
}
md += `\n## Summary\n\n| Role | journey-ready | thin-card | dead-tile |\n|---|---|---|---|\n`;
for (const role of Object.keys(byRole).sort()) {
  const c = (v) => byRole[role].filter((r) => r.ready === v).length;
  md += `| ${role} | ${c('journey-ready')} | ${c('thin-card')} | ${c('dead-tile')} |\n`;
}

writeFileSync(R('../docs/operations/ROLE_DEPTH_MATRIX.md'), md);
console.log(`wrote ROLE_DEPTH_MATRIX.md — ${rows.length} features across ${Object.keys(byRole).length} roles`);
```
