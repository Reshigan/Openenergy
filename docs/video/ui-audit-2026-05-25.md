# UI Audit — Corporate Video, Phase 1

**Date:** 2026-05-25
**Audience:** UN + Global ESCO Network (continuous-use product video, 15:00 master)
**Method:** Playwright walker (`tests/video/audit.spec.ts`) logged in as each of 9 personas, captured 45 full-page screenshots at 1920×1080 against `https://oe.vantax.co.za`. Stored at `docs/video/audit-shots/` (gitignored). Reviewed against a 13-point checklist (12 UX + 1 brand-match-to-deck axis).

**One-line read:** The platform is structurally ready to film. Three issues will visibly degrade the master video if not fixed before recording: (1) a cookies popover that intrudes on every shot, (2) sparse demo data on three high-camera-time launch boards (trader / offtaker / lender), and (3) one live 500. Brand drift vs the deck is a separate, larger conversation we should resolve before lock.

## 2. What's already production-grade for camera

For balance — much of the platform films well as-is:

- **Per-role launch boards** are visually consistent, dense without being cluttered, and the "Good morning, <name>" line is a strong scene-opener.
- **Trading terminal** has a real order book with bid/ask depth, live VWAP, fills table, and a recent-prints sparkline. This is the platform's hero screen and looks the part.
- **IPP-developer launch board** (Johan) has the richest seeded data — 4 projects, real "Awaiting your action" pipeline, suggested next steps with `WHY` reasons. Reads as a real workflow, not a demo.
- **Regulator workstation chrome** is correct (tabs: Surveillance triage / Licence actions / Enforcement events / Audit & compliance) — only the data inside the first tab is empty.
- **Esums fleet view** below the broken header — site cards render real OPEX/availability/MTTR with health badges and active alerts. Excellent for the IPP scene.
- **Color/typography** is consistent: navy/emerald/cream system across roles. Sidebar density and information hierarchy are professional.

## 3. Brand-match against the deck (separate finding)

The deck (`Consolidated_Energy_Cockpit_Corporate_2.pdf`) positions the product as:

- **Logo:** "LTM Energy" wordmark
- **Product:** "Consolidated Energy Cockpit"
- **Palette:** deep navy `#0f1c2e` + emerald `#1a7a3a` + warm gold `#c89432`
- **Typography:** Inter-family sans-serif

The deployed platform reads:

- **Logo:** "OPEN ENERGY" wordmark (top-left + login)
- **Product:** "South Africa's unified energy exchange" (login tagline)
- **Palette:** very similar — navy/cream/teal/emerald (close match)
- **Typography:** Inter-family sans-serif (match)

**Three options:**

1. **Keep "Open Energy" branding in the recording** and let the voice-over position it as a *deployment* of the LTM Energy / Consolidated Energy Cockpit product line. The login becomes a real-world reference customer. Zero code change.
2. **Brand-swap the SPA chrome** for the master cut only — `OPEN ENERGY` → `LTM ENERGY` wordmark + cockpit-themed tagline behind a `?brand=ltm` query string. Half-day of work.
3. **Full rebrand of the deployed platform** (risky, out of scope for video).

**Recommendation:** Option 1 — keep the deployment branding, let the V/O explain the product line. This also makes the video reusable for any future deployments.

## 4. Seed migration plan (`079_video_demo_seed.sql`)

Targets the P0 + P1 data gaps above. All inserts `OR IGNORE` (idempotent against the existing demo seed). Anonymized institutional naming per spec convention ("Solar IPP 01 — Northern Cape" etc.). No PII, no real company names.

**What it seeds:**

- **Trader:** 6 open orders (3 buy / 3 sell across solar / wind / hybrid), 3 partial-fill matches dated today, 1 fully-settled match.
- **Offtaker:** 4 active PPAs linking the seeded offtaker tenant to 4 IPP projects, each with a different tenor / strike / energy type.
- **Lender:** 3 senior facilities, 6 covenants (LCR / DSCR / availability) across the same projects.
- **Regulator:** 4 surveillance triage decisions (2 resolved, 2 open) with structured reason codes.
- **Carbon fund:** 3 registry positions + 1 issuance event.
- **Support:** 2 open tickets + 1 resolved + 1 SLA breach.
- **Admin persona display name:** rename `admin@openenergy.co.za` from "System" → "Sipho Mbeki / Platform Admin" so Act 3's opening shot doesn't read "Good morning, System".

**What it does NOT seed (out of scope for camera):**

- Order book depth — the trading terminal already renders real depth from the OrderBook DO based on the open orders we seed, so no need to write directly to that surface.
- Esums sites — the existing migration `074_esums_demo_seed.sql` already seeds 4 sites. The visible "0 sites" header is a query bug, not a data gap; we'll fix that in code, not seed.
- Cross-cutting / public pages (status, legal, audit) — these are already sufficient.

## 5. Cookie popover fix

The privacy-consent popover (`pages/src/components/CookieConsentBanner.tsx`) renders whenever `localStorage['cookieConsent']` is absent. For recording, we'll add to the recording suite's shared `seedToken` helper:

```ts
await page.addInitScript(() => {
  localStorage.setItem('token', '...');
  localStorage.setItem('cookieConsent', 'accepted');
  localStorage.setItem('cookieConsentTime', new Date().toISOString());
});
```

Zero production code change. Banner remains exactly as shipped for real users.
